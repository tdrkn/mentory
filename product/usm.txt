# User Stories (Mentory, MVP baseline)

Источник: `/Users/arxungo/Downloads/Отчет.pdf` (разделы "User Story (MVP)", "Сценарии использования", "FR/NFR").

## 1) Пользователь (общие)

### US1 — Регистрация и вход
Как пользователь, хочу создать аккаунт и войти в систему, чтобы пользоваться платформой.

Критерии приемки:
- Регистрация по email + пароль, вход в систему после регистрации.
- Ошибка при неверных учетных данных.
- Восстановление доступа по email.
- После входа пользователь попадает на главную/в личный кабинет.

### US2 — Профиль пользователя
Как пользователь, хочу заполнить структурированный профиль, чтобы консультация была полезной.

Критерии приемки:
- Есть обязательные и опциональные поля.
- Для роли mentor и mentee набор полей отличается.
- При незаполненных обязательных полях сохранение блокируется.
- Профиль можно редактировать, после сохранения есть подтверждение.

### US3 — Чат между участниками
Как пользователь, хочу общаться в чате для уточнения деталей встречи.

Критерии приемки:
- Отправка и получение текстовых сообщений в общем диалоге.
- Видны время отправки и статус прочтения/доставки.
- История сохраняется между сессиями входа.

### US4 — Видеосессии
Как пользователь, хочу участвовать в видеосессиях, чтобы занятия проходили эффективно.

Критерии приемки:
- Есть возможность прислать/получить ссылку на видеовстречу.
- Ссылка сохраняется в истории диалога.
- Оба участника могут открыть/скопировать ссылку.

## 2) Менти

### US5 — Поиск ментора
Как менти, хочу искать менторов по критериям, чтобы быстро находить релевантных.

Критерии приемки:
- Доступен список менторов.
- Работают фильтры минимум по специализации/направлению и цене.
- При отсутствии результатов показывается подсказка изменить фильтры.

### US6 — Просмотр профиля ментора
Как менти, хочу изучить профиль ментора, чтобы принять решение о записи.

Критерии приемки:
- Видны экспертиза, опыт, формат работы, стоимость.
- Видны отзывы и рейтинг.
- Из профиля доступно действие "записаться/отправить запрос".

### US7 — Запись к ментору
Как менти, хочу записаться к ментору, чтобы получить консультацию по запросу.

Критерии приемки:
- Выбор свободного слота в расписании.
- В заявке указывается тема/цель консультации.
- Запись получает статус (например, "на согласовании").
- Менти получает результат рассмотрения заявки.

### US8 — Оплата сессии
Как менти, хочу оплачивать сессии, чтобы получать услуги ментора.

Критерии приемки:
- Перед оплатой показаны сумма, длительность, комиссии/налоги, правила возврата.
- Оплата проходит выбранным способом.
- Обрабатываются ошибки оплаты.
- После успеха есть подтверждение/чек.
- Повторная оплата одной и той же сессии невозможна.

### US9 — Отзыв о менторе
Как менти, хочу оставлять оценку и отзыв, чтобы делиться мнением.

Критерии приемки:
- Отзыв доступен только после завершенной сессии.
- Оценка 1..5 и комментарий сохраняются.
- Отзыв отображается в профиле ментора.

## 3) Ментор

### US10 — Управление тарифами
Как ментор, хочу устанавливать тарифы для разных форматов работы, чтобы управлять доходом.

Критерии приемки:
- Ментор может создать/изменить тариф.
- Некорректная стоимость не сохраняется.
- Актуальные тарифы видны менти в профиле ментора.

### US11 — Подтверждение/отклонение заявок
Как ментор, хочу принимать или отклонять заявки, чтобы управлять нагрузкой.

Критерии приемки:
- Есть список входящих заявок.
- По заявке видны цель и контекст менти.
- Решение (принято/отклонено) отправляется менти, при отклонении можно добавить комментарий.

### US12 — Управление календарем
Как ментор, хочу гибко управлять календарем сессий.

Критерии приемки:
- Ментор может добавлять/удалять доступные слоты.
- При бронировании слот блокируется для других.
- Изменения расписания сохраняются и корректно отображаются.

### US13 — Автоматическое начисление оплаты
Как ментор, хочу получать оплату автоматически после сессии.

Критерии приемки:
- После выполненной сессии начисляется сумма за вычетом комиссии.
- Для одной сессии начисление не дублируется.
- При сбое фиксируется ошибка и отправляется уведомление.
- В личном кабинете доступна история начислений.

## 4) Администратор

### US14 — Обработка жалоб
Как администратор, хочу обрабатывать жалобы пользователей, чтобы улучшать клиентский опыт.

Критерии приемки:
- Пользователь может создать жалобу на пользователя/контент/сессию.
- Жалоба попадает в админ-панель с контекстом.
- Жалоба проходит статусы "Решено"/"Отклонено".
- Действия администратора логируются.

### US15 — Модерация контента
Как администратор, хочу модерировать контент пользователей.

Критерии приемки:
- Администратор видит список контента (профили, описания, отзывы).
- Скрытый контент недоступен обычным пользователям.
- Пользователь получает уведомление о модерации и причине.
- Есть журнал действий модерации.

### US16 — Верификация регалий ментора
Как администратор, хочу верифицировать документы и регалии менторов.

Критерии приемки:
- Ментор отправляет PDF-документы на проверку.
- В админ-панели есть очередь заявок на верификацию.
- Администратор может подтвердить/отклонить с комментарием.
- Решения логируются; при необходимости возможна блокировка ментора.

## Примечание по объему MVP
- В этом baseline оставлены только US1-US16 (как в отчете).
- Функции вроде "избранное", расширенной аналитики и генерации отчетов относятся к post-MVP/backlog.
